Why is Anthropic disabled?

Browsers have a built-in security rule called CORS (Cross-Origin Resource Sharing) that blocks web pages from making requests to a different domain than the one they loaded from. When your browser tries to call api.anthropic.com directly, it first sends a "preflight" check asking the API if cross-origin requests are allowed — and since Anthropic's API isn't configured to say yes to random websites, the browser shuts the request down before it even gets a response. But even if you somehow worked around that, there's a second problem: your API key would have to live inside your JavaScript code, which is fully visible to anyone who opens DevTools. That's like taping your credit card number to the outside of your laptop. The right solution is to put a backend server in the middle — your browser talks to your server, your server (which no one else can peek inside) holds the API key and makes the actual call to Anthropic, and the response gets passed back safely.
